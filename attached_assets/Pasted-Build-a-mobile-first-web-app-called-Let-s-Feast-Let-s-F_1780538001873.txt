Build a mobile-first web app called Let’s Feast.

Let’s Feast is a multi-channel food, delivery, restaurant discovery, nightlife, catering, grocery, and food experience platform.

For this first MVP, focus only on the customer-facing app.

Use a clean, premium, modern design with a light theme, red and gold accents, rounded cards, smooth spacing, and a food-tech feel. The app should feel simple and trustworthy like Amazon, but focused on local restaurants and food experiences.

The app should include these customer-facing sections:
1. Home
2. Explore
3. Delivery
4. Restaurant Menu
5. Cart
6. Checkout
7. Profile
8. More

Use mock data for restaurants, menu items, carts, Feast Windows, One Drink venues, Reserva restaurants, catering packages, grocery products, and Feast 360 events.

Do not connect real payments yet. Use mock checkout.

Create the following app flow:

OPENING SCREEN:
When the user opens the app, show:
- Let’s Feast logo
- Tagline: “Food. Experiences. Delivery. All in one place.”
- Button: “I’m New Here”
- Button: “Sign In”
- Link: “Forgot Password?”

SIGN UP:
When the customer clicks “I’m New Here,” show a signup form with:
- Full name
- Phone number
- Email
- Password
- Confirm password
- Zip code or location
- Optional referral code
- Button: “Create My Feast Account”

SIGN IN:
Create a sign-in page with:
- Email or phone number
- Password
- Forgot Password link
- Continue with Google placeholder
- Continue with Apple placeholder
- Button: “Sign In”

ONBOARDING:
After signup, show onboarding questions so Feast can personalize the customer experience.

Ask:
1. What kind of food do you like?
Options: Mexican, Puerto Rican, American, Italian, Chinese, Thai, Soul Food, Mediterranean, Burgers, Pizza, Seafood, Vegan, Desserts, Coffee, Breakfast

2. Do you have any allergies?
Options: Peanuts, Tree nuts, Dairy, Shellfish, Gluten, Soy, Eggs, Fish, Sesame, None

3. Any dietary preferences?
Options: Vegetarian, Vegan, Halal, Kosher, Gluten-free, Low-carb, High-protein, No pork, No beef, No preference

4. How far do you want to explore?
Options: Within 1 mile, Within 3 miles, Within 5 miles, Within 10 miles, Show me the whole city

5. What are you usually looking for?
Options: Fast delivery, Deals, Pickup, New restaurants, Healthy food, Family meals, Late-night food, Catering, Date night, Events

Button: “Build My Feast Experience”

FEAST WINDOW FEATURE:
Add a customer-facing countdown timer for restaurant-controlled Feast Windows.

Do not use the backend word “batching” on the customer side.

When a restaurant has a Feast Window active, show:
- “Feast Window Open”
- Delivery window time, example: “Delivery between 12:30 PM – 1:00 PM”
- Spots filled, example: “7 of 12 spots filled”
- Countdown timer, example: “Order in 04:32 and save $3”
- Button: “Join Window & Save”
- Secondary option: “Continue with Regular Delivery”

The countdown timer should appear:
- On restaurant cards
- On the restaurant menu page
- In the cart
- On checkout

If the timer expires, show:
“This Feast Deal expired. Choose the next Feast Window or continue with regular delivery.”
